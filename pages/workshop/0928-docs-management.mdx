## Docs Website Overview

Our docs website is built with **Nextra**, a static site generator for **Next.js**, and is hosted on **Vercel**. The site updates automatically whenever changes are pushed to the GitHub repository.

### GitHub for Version Control

GitHub allows team collaboration without conflict. We use **GitHub Pages** to host the docs, and the "Click to Edit" feature simplifies contributions, making it easy for anyone to update documentation without setting up a local environment.

> Start by exploring our [Docs Website](https://adastra-tw.github.io/docs/).

---
## Quick Guide: Edit on GitHub
1. Navigate to the page you want to edit on the [Docs Website](https://adastra-tw.github.io/docs/).
2. On the right side, click the **Edit this page** button.
3. You should be redirected to the corresponding page on GitHub.
4. Click the pencil icon to start editing. You can preview your changes by clicking the **Preview** tab.
5. Once you're done, click **Commit changes...** at the top right of the page. Then, click **Commit changes** again to confirm.
6. Your changes will be live on the Docs Website once the build is complete (usually within a few minutes).

## Markdown & MDX

### Markdown Basics

Markdown is a simple language for formatting text. Here are the essentials:

- **Headers**: `# Header 1`, `## Header 2`, `### Header 3`
- **Lists**: `- Item 1`, `- Item 2`, `- Item 3`
- **Links**: `[Text](https://example.com)`
- **Images**: `![Alt Text](https://example.com/image.jpg)`
- **Code**: `` `Inline Code` `` or ` ```Block Code``` `
- **Bold**: `**Bold Text**`  
- **Italics**: `*Italic Text*`

### MDX Basics

MDX enhances Markdown by allowing JSX components. Here’s an example of using a custom component in MDX:

```jsx
import CustomComponent from "../components/CustomComponent";

<CustomComponent />;
```

## Best Practices for Documentation

Writing good documentation is essential for clarity and understanding. Keep these tips in mind:

- Use **headings** to structure your content clearly.  
- Break up long text with **lists** or **bullet points**.  
- Use **images** or **diagrams** to clarify concepts.  
- Display code using **code blocks**.  
- Add **links** for external resources.

## Local Development: A Simple Example

Let’s walk through setting up a basic **Nextra** documentation site from scratch.

<Steps>
### Step 1: Clone the Docs Repository

First, create a new Next.js project and install Nextra:

```bash
git clone https://github.com/adastra-tw/docs.git
cd docs
pnpm install
```

### Step 2: Open the Project in VS Code and Start the Dev Server

Open the project in **Visual Studio Code** and start the development server:

```bash
code .
pnpm dev
```

### Step 3: Create Your First Markdown Page

Now, create a new markdown file under `pages`:

Inside `new-page.mdx`, add the following content:

```md
# Welcome to My Docs Site

This is an example of a simple documentation page using **Nextra**. You can add more content and customize this page using MDX components.

import Callout from '../components/Callout';

<Callout type="info">
  **Note**: This is a custom component.
</Callout>
```

Save the file and open `_meta.json` to add the new page:

```json
{
    // ...other pages,
    "new-page": "New Page"
}
```

### Step 4: View Your New Page

Navigate to `http://localhost:3000/new-page` to see your new page in action.

</Steps>

> For more details on customizing Nextra, check the official [Nextra Docs](https://nextra.site/docs/docs-theme/page-configuration).
